# 🚀 极致纯净 Nix DevOps 系统指南

恭喜！你现在使用的是一套经过深度重构的“双擎驱动”操作系统环境：
它在前端拥有 **ML4W 华丽动态的桌面环境**，而在引擎盖下，则跳动着一颗 **DevOps 级别的极客终端心脏**。

本指南将帮助你快速熟悉这套系统的核心功能、目录结构以及日常使用技巧。

---

## 1. 核心架构与哲学 (Nix Modular)

这套系统采用了**“极致纯净的模块化分离架构”**。所有配置文件都存放在 `~/home_env_dotfiles-ML4W` 中。它的设计哲学是“互不干扰，高内聚”：

- **`nix/modules/core/`**: 系统基础组件（`system-utils.nix`、字体等），包含 eza, bat, btop 等现代 Rust 工具。
- **`nix/modules/dev/`**: 开发组件（`dev-tools.nix`），包含各种开发工具和专门为 DevOps 定制的 Neovim 环境（YAML、Shell 语法支持）。
- **`nix/modules/desktop/`**: 图形化组件（`gui-apps.nix`、`theme.nix`），所有的桌面应用、主题和热重载逻辑都在这里。
- **`nix/modules/shell/`**: 终端组件（`zsh.nix`、`shell-common.sh`），负责你的终端体验和防护盾。

## 2. 终端防护与功能 (The Error Shield)

你的终端（Terminal）拥有极其智能的环境感知能力：
1. **容器隔离盾 (Container Shield)**：当你在 Distrobox 或 Docker 容器里打开终端时，系统会自动屏蔽那些在容器里跑不通的高级别名（比如 `z` 或 `cd` 的重定向），防止容器崩溃。
2. **SSH 自动降级**：如果你通过 SSH 远程连入这台电脑，Zellij（终端多路复用器）不会强行自动启动，避免终端嵌套卡死。
3. **极速自动补全**：删除了臃肿卡顿的全盘搜索插件，为你恢复了速度最快、手感最好的**灰字幽灵提示（Zsh Autosuggestions）**，只要按下键盘 `→` 键即可直接补全。

## 3. 日常维护命令 (How to Update)

这套环境是“配置即代码（Infrastructure as Code）”的。这意味着**所有改动必须先写进文件，再执行命令生效**。

| 命令缩写 | 完整命令 | 作用 |
| :--- | :--- | :--- |
| **`hms`** | `nix run home-manager/master ...` | **重载并应用所有配置更新**。当你修改了仓库里的 `.nix` 文件后，必须敲 `hms`，新的改动才会生效。底层采用平滑热更技术，更新时绝不卡死屏幕。|
| **`nix-clean`** | `nix-collect-garbage -d` | **清理系统垃圾**。Nix 每次更新都会保留旧版本的备份，当你觉得系统稳定不需要回滚时，运行此命令可以瞬间清理出大量硬盘空间。|

## 4. 救命快捷键与自救指南

如果你不小心修改了快捷键或者配置文件，导致桌面崩溃或者卡死，记住这三招：

### 🚑 桌面卡死（红条报错）
- **`Super + Shift + R`**：瞬间重载 Hyprland 配置。我们已经为此快捷键加入了通知弹窗。如果配置文件修好了，按它就能立刻消除屏幕红条并恢复控制。

### 🚨 完全黑屏失控（终端自救）
如果图形界面彻底崩溃：
1. 按下 **`Ctrl + Alt + F3`** 切换到纯黑白文本模式（TTY）。
2. 输入你的用户名和密码登录。
3. 进入配置文件夹（`cd ~/home_env_dotfiles-ML4W`），用 `git status` 看改错了什么，用 `git checkout -- .` 撤销你最近的错误改动。
4. 敲击 **`hms`** 重新部署正确的配置。
5. 按下 **`Ctrl + Alt + F1`**（或 F7）回到图形界面，满血复活！

## 5. 核心工具与快捷命令

我们在系统里预装了大量代替传统工具的“现代版本”：

| 旧工具 (已被替换) | 现代版本 (你现在在用的) | 特点与使用方法 |
| :--- | :--- | :--- |
| `ls` | **`eza`** | 输入 `ls` 会自动调用 `eza`，带有漂亮的图标、颜色和目录树形结构（`lt` 展开目录）。 |
| `cat` | **`bat`** | 输入 `cat 文件名` 会自动调用 `bat`，自带代码高亮和行号，阅读长文件体验极佳。 |
| `top` | **`btop`** | 终端最帅气的性能监控面板，直接输入 `btop` 查看 CPU/内存/网络使用情况。 |
| `vim` | **`nvim` (Neovim)** | 专为运维优化的轻量级编辑器，不仅秒开，还能在 Docker 容器内部穿透读取宿主机的配置文件！ |
